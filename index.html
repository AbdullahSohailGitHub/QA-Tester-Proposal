Screening Questions · Test Case Log

Answering your JD, point by point

Structured the way I structure everything I test — as a log, not a pitch.

TC-01Have you tested SaaS dashboards, reporting tools, or admin portals before?ANSWERED
Yes. I've run manual QA on Jetzy, a travel-and-social platform with a booking dashboard and admin-side reporting views, and Nooberly, a service-request marketplace with a multi-panel admin portal. Both involved dense tables, filters, and status-driven views close to what a multi-location retail dashboard needs.
TC-02How do you test when there's a walkthrough/spec but no mature QA process yet?ANSWERED
I treat the walkthrough as the source of truth for intended behavior, then build a lightweight test plan around the core workflows it covers — not a heavyweight formal suite. I test happy paths first, then deliberately break them: empty states, extreme values, slow loads, and mismatched filters.
TC-03What do you include in a good bug report?ANSWERED
Steps to reproduce, expected result, actual result, a screenshot or Loom, and the user impact — is this cosmetic, confusing, or blocking. See the sample report below.
TC-04How do you separate a functional bug from a UX issue or unclear requirement?ANSWERED
I ask one question: does it break the intended behavior, or does it just confuse the user while still working correctly? Both get logged, but tagged separately — Bug vs UX/Confusion vs Needs Clarification — so your team can prioritize fixes over debates.
TC-05Have you helped create or improve a QA checklist/process before?ANSWERED
Yes — I've put together reusable checklists after ad-hoc testing rounds so future testing didn't start from zero. That's part of what I'll hand back at the end of this trial.
TC-06Comfortable using Loom or screenshots? Available for the 10-hour trial?PASS
Yes to both — Loom for anything stateful or hard to capture in a still image, screenshots for everything else. I can start the 10-hour trial this week.
